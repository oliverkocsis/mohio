# Problem

## 1

Typical software development follows the following course of action - hopefully iteratively - 
1. Understand user needs
1. Define problem statement
1. Design solution
1. Write source code, 
1. Validate and verify
1. Use the solution

At every stage, there are numerous people involved using various communication tools resulting in a huge amount of communication errors. So the original user needs and the final solution usually falls far from each other. 

## 2 
Software development requires three essential roles and artifacts
- Product Manager who writes Requirements, because developers need to know what to do
- Developers, who writes Source Code, because machines need to know what to do
- Quality Assurance does Verification because the Product Managers writes bad descriptions and Developers add bugs to the Source Code.

In a perfect world, quality assurance would be obsolete. So why do we need still need that? Because compiling the requirements into source code is done by humans, and humans, by nature do mistakes. Noone verifies the machine code, because it is compiled from the source code using sophisticated algorithms. Hence we need to close the gap between requirements and source code. The requirement shall be compiled to source code by machines.
