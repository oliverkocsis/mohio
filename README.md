# Mohio

Mohio is an open source knowledge workspace for small teams. It combines fast note capture, structured team documentation, and AI-assisted organisation while keeping content stored as plain Markdown in a Git-backed workspace.

The goal is simple: make team knowledge easy to capture, refine, and share without forcing every contributor to think in Git, file structures, or documentation bureaucracy.

## Why Mohio

Teams often choose between two imperfect options:

- polished SaaS knowledge tools that are powerful but heavy,
- plain-text and Git-based workflows that are flexible but too technical for most teammates.

Mohio is designed to sit between those extremes. It aims to feel lightweight and approachable while keeping the portability, transparency, and ownership of Markdown files in version control.

## What Mohio Focuses On

- Fast personal capture through a scratchpad-first workflow
- Reviewable AI assistance for organising and improving shared knowledge
- Clear publishing and history for team documentation
- Markdown as the durable source format
- A familiar document workspace instead of a database-first interface

## Project Status

Mohio is in an early design and prototype stage. The repository currently focuses on product direction, documentation, and a browser-based v1 demo.

## Design Principles

- Git stays in the background for everyday users
- Shared knowledge should feel simple, explicit, and trustworthy
- AI should assist with structure and review, not silently publish changes
- The workspace should support connected knowledge without becoming hard to navigate
- Documentation should remain portable and easy to own outside any single platform

## Who Mohio Is For

Mohio is currently aimed at small teams that want a lightweight, document-first workspace for strategy, product, engineering, design, and operational knowledge.

It is especially relevant for teams that:

- prefer open formats over proprietary document storage,
- want AI assistance without giving up human review,
- need a simpler alternative to heavyweight wiki platforms,
- have at least one technical team member who can help with initial setup.
